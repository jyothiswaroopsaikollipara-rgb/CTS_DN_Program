# Module 1 - Design Patterns and Principles

## Overview
This module focuses on understanding object-oriented design principles and commonly used design patterns to develop maintainable and scalable software systems.

---

## Mandatory Exercises

### 1. Singleton Pattern
**Objective:**
Implement Singleton Design Pattern to ensure that only one object of a class is created.

**Concepts Used**
- Private Constructor
- Static Variable
- Static Method
- Lazy Initialization

**Files**
- Singleton.java
- SingletonMain.java

**Applications**
- Database Connections
- Logger
- Configuration Manager

---

### 2. Factory Method Pattern

**Objective:**
Implement Factory Method Pattern to create objects without exposing object creation logic.

**Concepts Used**
- Interface
- Polymorphism
- Factory Pattern

**Files**
- Vehicle.java
- Car.java
- Bike.java
- VehicleFactory.java
- FactoryMain.java

**Applications**
- Payment Gateways
- Notification Systems
- Database Drivers

---

## Additional Exercises

### SOLID Principles

- Single Responsibility Principle (SRP)
- Open Closed Principle (OCP)
- Liskov Substitution Principle (LSP)
- Interface Segregation Principle (ISP)
- Dependency Inversion Principle (DIP)

---

## Learning Outcomes

- Understand object-oriented design principles.
- Apply design patterns to real-world scenarios.
- Develop maintainable and extensible code.
- Reduce coupling and increase flexibility.
